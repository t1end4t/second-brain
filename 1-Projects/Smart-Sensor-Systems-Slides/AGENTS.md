## ⚠️ CRITICAL: Slide Generation Rule

**Each slide MUST be generated by its own `.mjs` file, producing its own `.pptx` file.**

```
slide1.mjs  →  slide1.pptx   (1 slide, 1 presentation)
slide2.mjs  →  slide2.pptx   (1 slide, 1 presentation)
slide3.mjs  →  slide3.pptx   (1 slide, 1 presentation)
...
```

**NEVER** write multiple slides into a single `.pptx`. Each `.mjs` file must:
1. Create a **new** `PptxGenJS` instance
2. Add **exactly one** slide
3. Write to a **unique** `.pptx` file named after the slide (e.g., `slideN.pptx`)
4. Use `pres.writeFile({ fileName: "slideN.pptx" })` — never reuse a shared output name

This per-slide approach allows:
- Independent review and iteration per slide
- Easy insertion/deletion without regenerating the entire deck
- Parallel generation if needed

## Tech Stack
- **Runtime:** Node.js (ES modules — `.mjs` files)
- **Library:** `pptxgenjs` (see `package.json`)
- **Run:** `node slideN.mjs`

## Design System (Consistent Across All Slides)

### Color Palette
| Role | Hex | Usage |
|------|-----|-------|
| Background | `0F1C2E` | Slide background |
| Primary accent | `0891B2` | Headers, badges, accent bars |
| Secondary accent | `F59E0B` | Warnings, trade-offs, highlights |
| Card fill | `1A2D47` | Content boxes |
| Text primary | `FFFFFF` | Main titles |
| Text secondary | `CBD5E1` | Body text |
| Text muted | `94A3B8` | Subtitles, captions |
| Text dim | `64748B` | Footer notes |
| Border/divider | `1A3A5C` | Lines, table borders |

### Layout
- **Left accent bar:** 0.12" wide, cyan (`0891B2`), full height
- **Content area:** starts at x=1.2", width=7.6"
- **Section badge:** top-left, pill shape, shows "N / M" format
- **Takeaway bar:** bottom of slide, cyan (`0891B2`), 1-line summary

### Typography
- **Titles:** Georgia, bold, white, 32-42pt
- **Section headers:** Calibri, bold, cyan, 12-14pt
- **Body text:** Calibri, 11-15pt, `CBD5E1`
- **Italics:** Calibri, muted colors for captions/notes

### Slide Types
1. **Title slide** (Slide 1): Large centered title, course info, no section badge
2. **Content slide** (Slides 2+): Section badge, title, content boxes, takeaway bar
3. **Table slide**: Structured table with header row
4. **Comparison slide**: Side-by-side or multi-column layout

## Workflow

### When asked to create slides:
1. Read `Chapter1-Outline.md` for the slide content plan
2. For each new slide:
   - Create `slideN.mjs` (N = next available number)
   - Follow the design system above
   - Write to `slideN.pptx`
   - Run `node slideN.mjs` to verify output
3. Update `TODO.md` to mark slide as done

### When asked to modify slides:
1. Edit the specific `slideN.mjs` file
2. Run `node slideN.mjs` to regenerate
3. Do NOT regenerate slides that weren't changed

### When asked to add/delete/reorder slides:
1. Adjust numbering if needed (slideN.mjs → slideN+1.mjs, etc.)
2. Update `Chapter1-Outline.md` to reflect changes
3. Generate only the affected slides (not the entire deck)
