# Personal Productivity Instructions

This area manages the user's planning and review workflow.

## Read First

- Read `Review.md` when the user asks what to do today, what changed, or how to plan.
- Read `TODO.md` before changing priorities.
- Read `Overview.md` before changing the system.

## Write Rules

- Put recurring workflow changes in this area.
- Put concrete project tasks in the relevant `1-Projects/*/TODO.md`.
- Put learning topics or ongoing interests in their own `2-Areas/*` folder.
- During review, update task state instead of only giving advice.
- Keep the system lightweight. Do not add daily logs, inboxes, or new folders unless the user asks or the current workflow clearly needs them.
