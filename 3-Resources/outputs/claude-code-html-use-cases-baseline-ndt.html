Baseline NDT × Claude Code

When HTML beats Markdown

Use HTML when Claude Code helps you inspect, compare, filter, review, or tune. Use Markdown when it records durable project memory.

Signal-rich output

Plots, PCA, heatmaps, waveforms, sensor maps → HTML.

Human decisions

Rotate/crop, LOI, masks, labels → interactive review UI.

Many comparisons

Models, conditions, plans, errors → cards/tabs/tables.

Project memory

Tasks, decisions, concise results → Markdown stays canonical.

Generate HTML when…

  • reader compares many visual states
  • answer contains plots/tables/heatmaps/PCA
  • filters, tabs, sliders, forms help
  • linear text may hide a bad choice
  • you need to stay “in the loop”

Keep Markdown when…

  • output is mostly prose
  • it is a task list or decision log
  • git diffs/search matter most
  • no visual comparison is needed
  • it updates vault state

Best HTML Use Cases

QA

Dataset QA dashboard

Counts, warnings, filters, drilldowns.

Raw TDMS inventory: file counts, short files, missing files, orientation, sensor/specimen/condition filters.
QA

Signal statistics report

Visual summaries scan faster than prose.

tdms_npy_stats_report.html: waveform IQR bands, peak timing scatter, TSNR, PCA, feature heatmap.
Review UI

Rotate/crop review

Visual decisions need visual context.

Raw peak map, corrected raster, rotation choices, crop rectangle, specimen mask overlay.
Review UI

LOI window selection

Waveform inspection plus saved choices.

Lift-off curves, candidate windows, chosen start/end, notes, CSV row export.
Review UI

Corrosion mask review

Spatial overlay matters more than text.

Feature map + corrosion mask + opacity slider + per-sensor tabs.
Modeling

Model result comparison

Metrics + errors need side-by-side layout.

1DCNN vs random forest vs logistic regression vs MLP: confusion matrices, calibration, per-condition errors.
Planning

Experiment plan explorer

Make agent plans readable and contestable.

2–4 preprocessing/modeling paths with tradeoffs, effort, risk, validation, recommendation.
QA

Error audit report

Find patterns in failures.

Sort/filter bad waveform length, low TSNR, bad crop, label mismatch, outlier embedding.

Reusable Patterns

One-file QA report

  • summary cards
  • warning panel
  • tabs for inventory/waveforms/PCA/outliers
  • embedded plots
  • exact reproduce commands

Human review interface

  • file list + filters
  • main plot/map preview
  • metadata + decision form
  • crop/opacity/LOI controls
  • CSV/JSON export

Side-by-side plan explorer

  • cards per approach
  • goal/steps/risk/validation columns
  • highlighted recommendation
  • copyable next prompt

Prompt: HTML report

Generate a single-file HTML review report, not just Markdown. Include summary cards, tabs, sortable tables, plots, warnings, and exact reproduction commands. Optimize it for quickly deciding what to inspect next.

Prompt: Review UI

Generate a browser-based HTML review UI with filters, visual previews, decision controls, and CSV/JSON export. Keep it single-file unless external assets are already produced by the pipeline.

Bottom line

In the NDT baseline project, HTML is best when Claude Code helps you see the data or make decisions from many signals. Markdown is best when Claude Code helps you remember the decision afterward.
