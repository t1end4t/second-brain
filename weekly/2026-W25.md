# Weekly Plan — 2026-W25 (June 15 – June 21)

## Top Priority
- Become a stronger AI-era researcher by protecting daily cycles of **Learn → Build → Think → Write**.

## Key Results
- [ ] Read + note 5 high-quality papers/articles.
- [ ] Run 5 small experiments/prototypes.
- [ ] Write 5 research journal entries.
- [ ] Produce 1 weekly synthesis: what I learned, what changed, what to test next.

## Daily Research System

### 1. Learn — 60–90 min
- Read 1 paper/article/technical blog.
- Track major AI developments.
- Save notes to knowledge system.
- Generate 3 questions:
  - What problem is solved?
  - Why does it matter?
  - What assumptions/limits exist?

### 2. Build — 1–3h
- Run 1 experiment, prototype, reproduction, or analysis.
- Prefer tiny daily progress over large unfinished plans.
- Examples: reproduce result, compare method, evaluate agent, create dataset, plot findings.

### 3. Think — 30 min
- Research journal:
  1. What did I learn?
  2. What surprised me?
  3. What questions remain?
  4. What should I test tomorrow?

### 4. Write — 30–60 min
- Compress knowledge into notes, summaries, reports, or beginner-level explanations.
- Writing goal: expose gaps, sharpen concepts, create reusable understanding.

## AI Usage Rules
- Use AI for summarizing, hypothesis generation, experiment design, code review, debugging, and assumption challenges.
- Do not outsource judgment, curiosity, taste, or problem selection.
- Always verify claims through source, code, or experiment.

## Schedule Shape
- **Morning:** Learn + questions.
- **Afternoon:** Build/test/analyze.
- **Evening:** Think + write + plan tomorrow.
